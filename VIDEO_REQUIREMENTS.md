# 视频提交要求

## 要求概述

根据 Project 3 提交要求，您需要提交一个视频：

### 视频内容要求

1. **演示者**：您或合作者出镜演示
2. **演示每个页面**：展示并讨论应用的每个页面
3. **突出亮点**：介绍您引以为豪的功能和特性
4. **游戏演示**：玩几轮游戏展示功能
5. **时长**：没有严格限制，但建议在 5 分钟以内

### 视频格式

- 可以上传到任何平台（YouTube, Vimeo, Google Drive, 等）
- 或者直接上传到 Canvas
- 确保链接可访问

## 建议的视频结构

### 1. 介绍 (30秒)
- 介绍自己和项目
- 简要说明项目是什么

### 2. 页面演示 (2-3分钟)
- **Welcome Page**：展示首页，点击链接
- **Rules Page**：展示规则说明
- **Game Selection Page**：展示游戏列表，创建新游戏
- **Game Page**：展示游戏界面，说明功能（Timer, Hint, Reset）
- **Scores Page**：展示排行榜

### 3. 功能演示 (1-2分钟)
- 创建游戏（Easy 或 Normal）
- 玩游戏：输入数字，展示验证功能
- 使用 Hint 功能
- 完成游戏，展示完成消息
- 查看排行榜更新

### 4. 亮点介绍 (30秒-1分钟)
- 强调您引以为豪的功能
- 例如：游戏生成器、实时验证、数据持久化等

## 录制建议

### 工具推荐
- **Mac**: QuickTime Player (内置屏幕录制)
- **Windows**: OBS Studio, Windows Game Bar
- **在线工具**: Loom, Screencast-O-Matic

### 录制技巧
- 确保屏幕清晰可见
- 说话清晰，语速适中
- 提前测试应用，确保流畅运行
- 可以准备简单的脚本或大纲

## 视频上传选项

1. **YouTube** (推荐)
   - 可以设置为未列出（Unlisted），只有有链接的人能看到
   - 易于分享和嵌入

2. **Google Drive**
   - 上传视频文件
   - 设置分享权限

3. **Canvas 直接上传**
   - 如果文件大小允许，可以直接上传

4. **其他平台**
   - Vimeo, Dropbox, OneDrive 等都可以

## 检查清单

- [ ] 视频包含所有要求的页面演示
- [ ] 展示了游戏玩法
- [ ] 突出了项目亮点
- [ ] 时长合理（建议 5 分钟内）
- [ ] 视频清晰，声音清楚
- [ ] 链接可访问
- [ ] 视频已上传或托管
